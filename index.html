Drupal

From version 6 to 8

Presentation by by Håvard Pedersen

History

Drupal started its life somewhere in 2000 as a custom Dutch online community / message board for a circle of friends, written by Dries Buytaert.

  1. Dorp.org
  2. Drop.org
  3. (Druppel)
  4. Drupal.org

The code was released as Drupal 1.0 15th January 20012.

Drupal is a Content
Management Framework

Drupal is like a Lego kit. Skilled developers have already made the building blocks - in the form of contributed modules - that you need to create a site that suits your needs[...]

Drupal development

  • Core vs Contrib
  • Backward compatibility for data, not code (proof)
  • Has traditionally been a Do-ocracy

A do-ocracy is an organizational structure in which individuals choose roles and tasks for themselves and execute them. Responsibilities attach to people who do the work, rather than elected or selected officials.

Drupal 6 features

  • Nodes (content types)
  • Comments
  • Custom URLs
  • Revisions
  • Taxonomy
  • User management (roles, permissions)
  • Regions and blocks
  • Menus

3rd party greatest hits

  • Content Construction Kit
  • Views
  • WYSIWYG

Drupal 7 changes

  • Do not break reverse proxies :)
  • Fieldable entities (nodes, users, taxonomy)
  • Entity, bundles, fields
  • Better administration UI
  • Database abstraction layer

With content nodes (an entity type), for example, you can generate bundles (subtypes) like articles, blog posts, or products.

Drupal 8 changes

  • Finally truly OO
  • PHPUnit
  • Doctrine
  • Fully reverse proxy compatible (w/fragments)
  • Components from Symfony 2 Framework
  • Comments on any entity (Inception)
  • Views in core
  • Twig templating language
  • WYSIWYG (10 years ago called...)
  • CMI (UUID, config files, etc)
  • Scotch initiative
  • Backbone.js (MVC in JS)

Full responsive mobile interface

Internationalization improved

Web Services and Context core Initiative
(WSCI = Whisky)

  • path
  • HTTP method
  • SSL
  • Mime-headers
  • Pluggable filters
  • Can serialize for you to JSON and XML

rest.module in core

  • GET, POST, PATCH, DELETE
  • Can even expose views as a collection!
  • JSON, XML and HAL (specific JSON)
